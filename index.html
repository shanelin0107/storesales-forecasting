This project addresses the Kaggle Store Sales - Time Series Forecasting competition: predict daily sales for every store–product pair in Corporación Favorita's 54-store Ecuadorian grocery chain. The task involves 1,782 concurrent time series and a 16-day forecast horizon (Aug 16–31, 2017).

The final pipeline combines 33 independent LightGBM models (one per product family) with recursive one-day-ahead forecasting — each model predicts the next day using its own prior predictions as lag features. A zero-sales post-processing step then zeros out discontinued or stockout series, eliminating systematic positive bias. Total improvement over the global baseline is 11%.

METHODOLOGY

Four-stage pipeline

01
FEATURE ENG.

40+ Feature Columns

Lag_1–63, lag_364/371, rolling means/stds, calendar, holiday (national/regional/local), oil price, promotions, earthquake dummy.

02
PER-FAMILY

33 Independent Models

One LightGBM per product family. PRODUCE, BEVERAGES, MEATS each have distinct scale and seasonality — separate models outperform one global model.

03
RECURSIVE

Day-by-Day Forecast

Predict Aug 16, feed it as lag_1 to predict Aug 17, and so on. Unlocks lag_7 — the strongest weekly cycle signal. Largest single gain: LB 0.418 → 0.397.

04
POST-PROCESS

Zero-Sales Filter

Store–family pairs with 3+ consecutive zero-sales days are set to 0. Eliminates positive bias from discontinued SKUs and stockout series. LB 0.397 → 0.385.
